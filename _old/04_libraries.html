Frameworks

Definition

  • Frameworks support fundamentally the way we use Javascript:
    • extensions of the javascript language with shortcuts and extras for loops, strings, arrays...
    • cross browser solutions for DOM handling or events
    • support a specific methodology like MVC
  • Some popular frameworks:
    • jQuery, Mootools, YUI, Dojo: DOM / events support with lots of extras like animations, AJAX support etc...
    • Prototype, Underscore: loose collection of language-enhancement functions (sometimes called salad frameworks)
    • AngularJS, React, Backbone, Knockout: MVC frameworks
    • jQuery mobile, Phonegap, XUI, Ionic: mobile frameworks
A few random remarks:
  • The term 'framework' is by no means official; it is widely used for a library which is considered 'fundamental', whatever that means.
  • Frameworks can be combined; a classic combination is jQuery (DOM) / RequireJS (script loader) / Backbone (MVC) / Underscore (template engine).
  • Mobile frameworks should be very lightweight and support mobile-typical features like swipes, device orientation...

Smallest framework ever

  • The smallest framework ever: 
    var $ = function (s) { return document.getElementById(s); };

Resource loaders

Definition

  • Resource loaders take care of when, how and in what order resources should be loaded.
  • Some popular Javascript/CSS loaders (check this overview):
    • Yepnope: conditional CSS and Javascript loader for polyfills
    • LazyLoad: ultrasmall and simple CSS and Javascript loader
    • RequireJS: focus on creating and using/loading Javascript modules
  • Other resource loaders:
    • SWFObject: currently by far the best method to embed Flash content
    • PXLoader: audio & images preloader for HTML5 games
    • PreloadJS: excellend all-round resource preloader (part of CreateJS)
Problems with loading scripts:
  • synchronous loading of files is limited to 2-4
  • loading a script blocks the UI, unless you put the <script> tag at the end
  • not all script are always needed, e.g. depending on user interaction
  • not all script are always wanted, e.g. browser specific fixes
Script loaders may
  • determine which script to load, and under what conditions
  • wait for a script to load before loading another, or instead allow parallel loading
Script loaders are generally faster, even when compared to combining all page scripts in one single file.

var camera, scene, renderer;

var init = function() {
  // variables
  var container = document.getElementById('container');

  // create scene
  scene = new THREE.Scene();

  // create camera
  camera = new THREE.OrthographicCamera(
    window.innerWidth / -2,
    window.innerWidth / 2,
    window.innerHeight / 2,
    window.innerHeight / -2,
    -2000,
    1000
  );
  camera.position.x = 200;
  camera.position.y = 100;
  camera.position.z = 200;

  // create grid
  var size = 500, step = 50;
  var geometry = new THREE.Geometry();
  for (var i = -size; i <= size; i += step) {
    geometry.vertices.push(new THREE.Vector3(-size, 0, i));
    geometry.vertices.push(new THREE.Vector3(size, 0, i));
    geometry.vertices.push(new THREE.Vector3(i, 0, -size));
    geometry.vertices.push(new THREE.Vector3(i, 0, size));
  }
  var material = new THREE.LineBasicMaterial({ color: 0x000000, opacity: 0.2 });
  var line = new THREE.Line(geometry, material);
  line.type = THREE.LinePieces;
  scene.add(line);

  // add cubes
  var geometry = new THREE.CubeGeometry(50, 50, 50);
  var material = new THREE.MeshLambertMaterial({ color: 0xffffff, shading: THREE.FlatShading, overdraw: true });
  for (var i = 0; i < 100; i++) {
    var cube = new THREE.Mesh(geometry, material);
    cube.scale.y = Math.floor(Math.random() * 2 + 1);
    cube.position.x = Math.floor((Math.random() * 1000 - 500) / 50) * 50 + 25;
    cube.position.y = (cube.scale.y * 50) / 2;
    cube.position.z = Math.floor((Math.random() * 1000 - 500) / 50) * 50 + 25;
    scene.add(cube);
  }

  // add ambient light
  var ambientLight = new THREE.AmbientLight(0x05);
  scene.add(ambientLight);

  // add directional light
  var directionalLight = new THREE.DirectionalLight(0x00ffcc);
  directionalLight.position.x = -0.8;
  directionalLight.position.y = 0.3;
  directionalLight.position.z = 0.5;
  scene.add(directionalLight);

  // create canvas
  renderer = new THREE.CanvasRenderer();
  renderer.setSize(window.innerWidth, window.innerHeight);

  // add canvas to container
  container.appendChild(renderer.domElement);

  // events
  window.addEventListener('resize', onWindowResize, false);
}

function animate() {
  requestAnimationFrame(animate);
  render();
}

function onWindowResize() {
  console.log('ok');
  camera.left = window.innerWidth / - 2;
  camera.right = window.innerWidth / 2;
  camera.top = window.innerHeight / 2;
  camera.bottom = window.innerHeight / - 2;
  camera.updateProjectionMatrix();
  renderer.setSize(window.innerWidth, window.innerHeight);
}

function render() {
  var timer = Date.now() * 0.0001;
  camera.position.x = Math.cos(timer) * 200;
  camera.position.z = Math.sin(timer) * 200;
  camera.lookAt(scene.position);
  renderer.render(scene, camera);
}

// start your engines
init();
animate();

Widgets, plugins & polyfills

Definitions

  • The difference is roughly as follows:
    • a widget is HTML, CSS and Javascript mixed in a more or less independent component, e.g. slider, calendar, tooltip...
    • a plugin is an (often third-party) extension to an existing framework, e.g. colorbox
    • a polyfill is a Javascript fix when a browser fails to implement an expected behaviour (also called shim or shiv), e.g. Webshims Lib for HTML5 or the notorious html5shiv for IE
In other words:
  • A widget contains HTML, CSS and Javascript
  • A plugin, polyfill or shim basically just contains Javascript
But then again, plugins may need some CSS too and polyfills may generate DOM elements too, so the division is not really sharp.

Border radius polyfill example

  • Border radius polyfill with Modernizr feature detection and html5shiv IE fix example: 
    <!DOCTYPE html>
<html>
<head>
  <title>Border radius shiv demo</title>
  <meta http-equiv="X-UA-Compatible" content="IE=EmulateIE7" />
  <meta charset="utf-8" />
  <!--[if lt IE 9]>
  <script src="js/html5shiv.js"></script>
  <![endif]-->
  <link href="css/style.css" rel="stylesheet" />
  <style>
    article p {
      margin: 0;
      padding: 0;
    }
    article {
      width: 400px;
      padding: 10px 20px;
      font: 16px Arial, sans-serif;
      background-color: #cc9;
      border-radius: 10px;
    }
  </style>
</head>
<body>
  <article>
    <p>Fusce vitae orci. Lorem ipsum dolor sit amet, consectetuer adipiscing  elit.</p>
  </article>
  <script src="js/modernizr-1.5.min.js"></script>
  <script src="js/jquery-1.4.2.min.js"></script>
  <script type="text/javascript">
    if (!Modernizr.borderradius) {
      $.getScript("js/jquery.corner.js", function () {
        $("article").corner();
      });
    }
  </script>
</body>
</html>
